# Principles — Always Loaded

These principles apply to ALL work. They are not context-dependent.

1. **Redistribution Over Removal** — When a capability appears misplaced, move it. Don't delete it. Deletion is justified only after confirming nothing in the system needs it anywhere.

2. **Explicit State Over Implicit Context** — Information that must survive a boundary must be written to an artifact. Context degrades at every hop. Artifacts persist.

3. **The Razor and the Pause** — Critical analysis (cut) and charitable interpretation (pause) are sequential operations. Both necessary, in order. OPTIMIZE then DELIBERATE then APPLY.

4. **Errors Are Curriculum** — Every failure carries information about the process itself. Name it. Ask: "What structural condition made this possible?" If generalizable, encode it.

5. **Correction Is Additive** — Don't suppress the original impulse. Add the complementary one. The result is stronger than either alone.

6. **Recursive Self-Application** — Validate a framework by applying it to itself. If it can't survive its own audit, it doesn't work.

7. **Tension as Generative Force** — Good architecture emerges from holding opposing forces simultaneously. Don't resolve tensions prematurely. The design that satisfies both poles is always stronger.

8. **Vocabulary Is Architecture** — Name patterns precisely. The name should describe the structural cause, not the symptom. Use the name consistently everywhere. A term must mean exactly one thing.

9. **Single Source of Truth** — Every artifact exists in exactly one canonical location. Copies are acceptable only as build outputs, never as sources.

10. **Progressive Disclosure** — Load only what's needed for the current phase. Front-loading all available information degrades quality through noise, not absence. The constraint is the context window; respect it.
